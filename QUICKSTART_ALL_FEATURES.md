## Getting Started

### Step 1: Install & Configure (2 minutes)

```bash
# 1. Navigate to dashboard directory
cd "c:\Users\regan\ID SYSTEM\axiom-x\constitutional-market-harmonics\dashboard"

# 2. Copy environment template
cp .env.example .env.local

# 3. Add your Anthropic API key (required for chat)
# Edit .env.local and replace:
#   ANTHROPIC_API_KEY=sk-ant-api03-your-actual-api-key-here
# Get key at: https://console.anthropic.com/

# 4. Install dependencies (first time only)
npm install
```

### Step 2: Start the Servers (1 minute)

```bash
# Terminal 1: Next.js Frontend (port 3000)
npm run dev

# Terminal 2: Express.js Backend (port 3001 + Socket.IO 12345)
npm run server
# Or manually: npm run build && npm start

# Wait for output:
# ‚úì Next.js ready at http://localhost:3000
# ‚úì Express backend listening on port 3001
# ‚úì Socket.IO listening on port 12345
```

### Step 3: Open Dashboard (2 minutes)

1. Open browser: **http://localhost:3000**
2. You'll see the **Overview Tab** with:
   - Your portfolio holdings (top)
   - Performance metrics (upper right)
   - Recent trades (middle)

   - News ticker (bottom)

3. **Try the Chat Feature:**
   - Look bottom-right corner for **floating chat button**
   - Click to open Claude AI chat overlay

## Common Questions & Troubleshooting

### "Why is my portfolio total different from expected?"

**Possible causes:**
1. ‚úÖ Mock data loaded (check if API keys configured in .env.local)
2. ‚úÖ Positions from database haven't synced (wait 5 seconds for refresh)
3. ‚úÖ Cash balance not included (check Portfolio tab ‚Üí Balances & Cashflow)

**Solution:**
- Check .env.local has valid API keys
- Wait for Socket.IO connection (green indicator in header)
- Check browser console (F12 ‚Üí Console tab) for errors
- Restart dashboard if no updates in 10 seconds

---

### "Chat interface isn't responding"

**Possible causes:**
1. ‚úÖ ANTHROPIC_API_KEY not configured in .env.local
2. ‚úÖ API key invalid or rate-limited (100 requests/minute limit)
3. ‚úÖ Network connectivity issue
4. ‚úÖ Backend server not running (express.js on port 3001)

**Solution:**
```bash
# 1. Verify .env.local has valid key
cat .env.local | grep ANTHROPIC_API_KEY

# 2. Check backend running
curl http://localhost:3001/api/dashboard

# 3. Check logs in Terminal 2 for errors
# Look for: "Error initializing Claude" or similar

# 4. Restart both servers
# Terminal 1: Ctrl+C then: npm run dev
# Terminal 2: Ctrl+C then: npm run server
```

---

### "Market data showing as 'N/A' or zero values"

**Possible causes:**
1. ‚úÖ Market data API keys not configured (ALPHA_VANTAGE_API_KEY, POLYGON_API_KEY)
2. ‚úÖ API rate limit exceeded (Free tier: 100-500 calls/day)
3. ‚úÖ Stock ticker symbol typo in database
4. ‚úÖ Market closed (weekends/holidays)

**Solution:**
```bash
# 1. Add market data API key to .env.local
ALPHA_VANTAGE_API_KEY=your_key_here

# 2. Restart dashboard (serverses need to reload .env)
# Ctrl+C on both terminals

# Terminal 1: npm run dev
# Terminal 2: npm run server

# 3. Check if mock data fallback working
# Open Global Markets tab ‚Üí should see "Mock Data" indicator if real data unavailable
```

---

### "Chart visualizations not rendering / showing blank"

**Possible causes:**
1. ‚úÖ Chart.js library not loaded (npm dependencies missing)
2. ‚úÖ No data available for chart (empty positions array)
3. ‚úÖ Browser JavaScript error (check console)
4. ‚úÖ Component error caught by ErrorBoundary

**Solution:**
```bash
# 1. Verify dependencies installed
npm list chart.js react-chartjs-2

# 2. If missing, reinstall
npm install chart.js@4.5.1 react-chartjs-2@5.3.1

# 3. Check browser console for errors
# F12 ‚Üí Console tab ‚Üí look for red error messages

# 4. Check that portfolio has positions
# Overview tab ‚Üí Portfolio Panel should show holdings
# If empty, that's why charts are blank
```

---

### "Socket.IO connection unstable / keeps reconnecting"

**Possible causes:**
1. ‚úÖ Network connectivity intermittent
2. ‚úÖ Backend server overloaded (too many requests)
3. ‚úÖ Firewall blocking port 12345
4. ‚úÖ CORS misconfiguration

**Solution:**
```bash
# 1. Check backend logs for connection errors
# Terminal 2 should show: "‚úì Socket.IO listening on port 12345"
# If not: backend crashed

# 2. Verify port availability
# Windows: netstat -ano | findstr :12345
# Should see a process listening

# 3. Check update interval (too frequent = server overload)
# In .env.local: UPDATE_INTERVAL=5000 (5 seconds minimum)

# 4. Try with fallback to polling (if WebSocket unavailable)
# Dashboard should auto-detect and switch to HTTP polling
```

---

### "Dashboard slow / takes 10+ seconds to load"

**Possible causes:**
1. ‚úÖ First load (Node modules compiling)
2. ‚úÖ API calls slow (external market data APIs)
3. ‚úÖ Database queries slow (large trade history)
4. ‚úÖ Too many real-time updates (UPDATE_INTERVAL too low)

**Solution:**
```bash
# 1. Check if first load (this is normal)
# Subsequent loads should be <3 seconds

# 2. Increase UPDATE_INTERVAL in .env.local
UPDATE_INTERVAL=10000  # 10 seconds instead of 5

# 3. Check database size
# If >100K trades: might slow down. Consider archiving old trades.

# 4. Disable mock data generation if not needed
ENABLE_MOCK_DATA=false

# 5. Check API response times
# Terminal 2 logs should show: "API latency: 500ms" etc
# If >2 seconds: API key rate limited or no connectivity
```

---

### Constitutional Scoring Explained (5 Yama Principles)

Position gets LOW score if:
- ‚ùå Company engages in fraud
- ‚ùå Invests in exploitative practices
- ‚ùå Overly concentrated in single asset
- ‚ùå High leverage (too much debt)
